# AtlasQuest
是有關世界旅行紀錄的系統，使用者旅行後，可在這個系統找出所有地點並記錄，透過統計圖表、成就紀錄，讓使用者可以在這個網站看到自己過往曾經去過的地方，或許類似遊戲成就一般。  



這個世界旅行紀錄系統的想法太棒了！它把「旅行」這件充滿感性與回憶的事，用「遊戲化（Gamification）」的理性數據與成就解鎖來包裝，會讓有收集癖、旅行癮的玩家（使用者）無可救藥地愛上黏著在你的系統上。

針對「世界地圖、百分比解鎖、遊戲成就、旅行里程」這些核心元素，我為你提供以下四個方向的系統名稱建議：

1. 探索與大航海風（強調世界地圖、100% 探索率）
這類名稱非常有 RPG 遊戲的既視感，打開系統就像打開迷霧地圖，讓人有一股「想把世界全部點亮」的衝動。

FogLifter / LiftTheFog (點亮世界 / 揭開迷霧)

概念： 源自遊戲中的「戰爭迷霧（Fog of War）」。每次旅行、每次登記，就像是在世界地圖上撥開一片迷霧。統計圖表就是你的「地圖探索率」。

Terra100 (地球百分解)

概念： Terra 是拉丁文的地球。名稱直指系統核心——你的目標就是解鎖這顆星球的 100%。聽起來像是一個高質感的 Web 專案。

AtlasQuest (地圖集遠征)

概念： Atlas（地圖集）加上 Quest（探險任務）。把厚重的世界地圖變成一場進度條大冒險，看著國家百分比上升就像在破關。

2. 遊戲化與稱號成就風（強烈推薦，契合解鎖美食、首都、里程功能）
如果你的系統重點在於「世界 10% 國家、5% 首都、美食達人」等自訂徽章與稱號，這類名字能直接給予使用者回饋感。

WanderBadge (浪遊徽章 / 萬里行章)

概念： Wander（漫遊）+ Badge（徽章）。每一次到訪、每一段里程、每一家踩點的餐廳，都能在系統裡轉化為一枚精美的實體化徽章。

GlobeTrophy / Globy (全球獎盃 / 地球小馬)

概念： Globetrotter（世界旅行者）與 Trophy（獎盃）的結合。縮寫成 Globy 聽起來親切又可愛，非常適合做成 APP 名字與吉祥物。

TitleHunter (旅遊稱號獵人)

概念： 像遊戲玩家一樣，為了解鎖「首都破壞者（到訪 5% 首都）」、「地表最強胃袋（百大美食踩點）」等稱號而前往世界各地的系統。

3. 里程與足跡風（強調移動距離、視覺化統計圖表）
如果你想突顯的是「旅行總里程數可繞地球幾圈」、「精美足跡連線圖」等強大數據 Dashboard 的質感。

OdysseyLog (奧德賽足跡 / 漫長壯遊)

概念： Odyssey 代表一場漫長、充滿冒險的史詩旅程。結合 Log（紀錄），給人一種大器、浪漫卻不失科技感的氛圍。

TraceGlobe (軌跡球 / 繪製地球)

概念： Trace（追蹤、描繪）+ Globe（地球）。用使用者的步伐在地球上連線，拉出壯觀的飛行與旅遊統計圖表。

MileStone (哩程碑)

概念： 雙關語。既是旅行的「哩程（Miles）」，也是人生中每一個值得紀念的「里程碑（Milestone）」。

4. 浪漫生活風格（適合社群分享、輕量精緻感）
這類名稱比較低調奢華，適合網頁版（Web）設計成非常有質感的個人數位旅遊客廳，分享給朋友看時很有面子。

MyBoundless (我的無界之地)

概念： 國界與距離都是虛擬的，在系統裡，使用者的足跡是無限延伸的。

RoamList (漫遊清單)

概念： 簡潔俐落。既是過往的紀錄，也是未來的 Wish List（願望清單），幫你勾選掉世界上每一個角落。

🛠️ 從前端/後端實作角度的額外聯想：
因為這個系統會大量用到 Geocoding（地理編碼）、SVG/Canvas 世界地圖、D3.js 或 ECharts 圖表：

如果網頁主視覺打算用一個可以拖曳旋轉的 3D 數據地球（WebGL），Terra100 或 TraceGlobe 會非常符合那種極致科技感的體驗。

如果系統偏向精緻的卡牌解鎖、像素風小徽章，WanderBadge 或 Globy 會讓介面設計的調性更活潑。
